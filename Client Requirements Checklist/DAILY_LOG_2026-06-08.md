---

**Problem #1 — Saving Colour Variants to Shopify Was Failing Silently**

The admin form had a toggle for "Colour Variants." When enabled, the user could type colour names and save. But reopening the product showed the toggle back to OFF — nothing was saved.

Root causes found:
- The PUT route (`/api/admin/products/[id]`) never read `hasVariants` or `variants` from the request body — so the server always treated it as a single-variant product.
- The form's variant UI included Price, Compare At, and Stock fields per colour row — but Shopify's variant API at the time didn't accept all those fields together, causing silent rejection.

**Fix:** Wired `hasVariants` and `variants` into the PUT route body destructure. Simplified the colour variant row to colour-name only (price shared from the main product price field at that stage).

---

**Problem #2 — "Option Does Not Exist" Error on Shopify API**

Once the data was flowing correctly, Shopify returned: *"Option does not exist."*

This happened because `productVariantsBulkCreate` requires the "Colour" option to already exist on the product before variants can be added. The two-step approach (`productOptionsCreate` then `productVariantsBulkCreate`) was failing because Shopify's `productOptionsCreate` errors were not being checked — if it failed silently, the next step always errored.

**Fix:** Replaced the entire two-step with a single `productSet` mutation, which atomically sets both `productOptions` and `variants` in one API call. This is Shopify's recommended approach for this scenario.

---

**Problem #3 — productSet Returning "Option Must Specify At Least One Value"**

After switching to `productSet`, the next error was: *"Option 'Colour' must specify at least one option value."*

The `productOptions` input was being sent as `[{ name: 'Colour' }]` without listing the actual colour names as values.

**Fix:** Updated the input to `[{ name: 'Colour', values: colourVariants.map(cv => ({ name: cv.colour })) }]`.

---

**Problem #4 — Stock Always Saving as Zero**

Variants were now creating successfully in Shopify, but inventory showed 0 for every colour. Two causes:

1. The form hardcoded `stock: 0` on every colour variant — the user had no way to enter stock per colour.
2. On subsequent saves (when variants already existed in Shopify), `updateAdminProduct` had no code path to update existing variants — it only ran `productSet` when `existingColourEdges.length === 0`. Any re-save of stock or price did nothing.

**Fix:**
- Added `stock` (and `price`) input fields per colour variant row in the admin form.
- In `updateAdminProduct`, added a second branch: when colour variants already exist, call `productVariantsBulkUpdate` to update prices, then call `setInventoryQuantity` for each variant matched by colour name (not by index, which is unreliable).
- Updated the `productSet` response query to return `selectedOptions` so variants are matched by colour name rather than array position.

---

**2. Replaced Plain Text Colour Input with Shopify-Style Colour Picker**

The original admin form used a plain text input where the user typed colour names manually (e.g. "Clear", "Red"). This was confusing and error-prone.

Replaced with a proper colour swatch picker:
- A "+  Add colour" button opens a dropdown panel
- Panel lists 16 preset colours (Red, Orange, Yellow, Green, Blue, Powder Blue, Pink, Peach, Magenta, Brown, Gold, Silver, Amber, Clear, White, Black) — each with a coloured circle swatch and a checkbox
- Selected colours appear as chips above the picker with their coloured dot
- Each selected colour row shows: swatch + name | Price ($) input | Qty input | Remove (×)
- A "Add custom colour" text input at the bottom of the dropdown for non-preset entries
- Clicking outside the dropdown closes it

---

**3. Fixed Storefront Colour Swatch Display**

The storefront `ProductInfo.tsx` was using `colorVariants.length > 1` to decide whether to show the colour picker — meaning a product with exactly 1 colour variant would never show the selector. Changed to `>= 1` in both `ProductInfo.tsx` and `lib/shopify.ts` (which also uses `> 1` to set `variantId: null`).

The storefront now correctly shows colour button selectors for any product with one or more colour variants, and marks out-of-stock colours as disabled.
