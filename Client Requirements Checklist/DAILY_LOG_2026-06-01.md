#### Shopping Cart — Shopify Connected (May 30)
The cart (`/crate`) is fully connected to Shopify's Storefront Cart API:

- Cart persists across page refreshes via `localStorage` (Shopify cart ID stored locally)
- On return visits, the cart automatically reconnects to the existing Shopify cart
- Adding/removing/updating quantities syncs to Shopify in the background (optimistic updates — UI always instant)
- Stock limits enforced everywhere: catalog, product detail page, cart drawer, and full cart page
- Rapid-tap debouncing prevents Shopify conflict errors
- Stale cart recovery: if the Shopify cart expires, the site creates a fresh one automatically on the next action

#### Bug Fixes Applied (May 30)
| Bug | Fix |
|---|---|
| Customers could exceed stock by tapping "Add to Crate" from catalog | Stock limit now enforced at every add point |
| Rapid + clicks sent overlapping Shopify requests → conflict errors | Debounced: UI updates instantly, Shopify gets one clean request |
| Stale cart ID kept triggering repeat errors | On sync failure, stale ID cleared and cart reconnected |
| + button on `/crate` page didn't cap at stock limit | Stock cap now enforced on the full cart page |
| Product detail quantity selector showed "1" even when item already in cart | Selector now reads the cart on load and stays in sync |
