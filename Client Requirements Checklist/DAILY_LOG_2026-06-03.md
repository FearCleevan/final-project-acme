## What We Worked On Today

### 1. Signs Page — Getting Scott's 3 Sign Products to Display
Scott sent photos and details of 3 antique enamel advertising signs (Cadbury's, James Hinks & Sons, Veritas Mantles) at $890 each. We set up everything needed to show them on the site:

- Created a new **Signs** collection in Shopify and added all 3 products to it
- Set the collection URL handle to `/collections/signs` so the website knows where to find them
- Updated the website code to recognize the "Signs" category and route those products to the dedicated `/signs` page
- Fixed an issue where the collection was not published to the Online Store — this was the main reason signs weren't showing up (Shopify only shares products with the website if they're marked as published)
- Signs now display correctly at `/signs` with all 3 photos, prices, and the "Add to Crate" button

### 2. Removed the Word "Advertising" from Signs Labels
Scott's direction was simply "Signs" — not "Advertising Signs." Cleaned this up in every place it appeared: the nav menu, the sidebar filter, the mobile drawer, the footer links, and the page title.

### 3. Admin Dashboard — Product Form Now Opens as a Full Page
Previously, when you clicked "Add Product" or clicked a product to edit it in the admin dashboard, a popup window (modal) would appear. This popup had two problems:
- The background page kept scrolling behind it
- You couldn't scroll inside the popup using the mouse wheel

We replaced the popup entirely. Now clicking **Add Product** takes you to `/admin/products/new` (a proper full page), and clicking any product takes you to `/admin/products/[id]` (its own page too). Both pages have:
- A breadcrumb at the top so you know where you are (e.g., **Products → New Product**)
- Discard and Save buttons that stay visible as you scroll
- After saving, the page automatically returns you to the products list
- **Draft saving** — if you're halfway through filling in a new product and your browser refreshes, your work is not lost. It saves automatically as you type.

### 4. Fixed Deployment Error on Vercel
The live Vercel deployment was failing to build because of a technical Next.js requirement — any page that reads from the URL (like checking if you came from the products list) must be wrapped in a special container called `<Suspense>`. Added this to both the New Product and Edit Product pages. Build now passes.

### 5. Small Fixes Along the Way
- Fixed the breadcrumb bar covering the admin header (it was sitting behind the top navigation instead of below it)
- Fixed the Signs sidebar filter value — a code formatter had quietly changed `'signs'` to `'sign'` (missing the 's'), which broke the filter
- Cleared the website's data cache so new Shopify products appear immediately instead of waiting 60 seconds
