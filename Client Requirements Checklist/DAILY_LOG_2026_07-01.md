## What Got Done Today

### 1. Admin Dashboard: Interactive Delivery Status Dropdown
We updated the internal Admin Orders page (`OrdersPage.tsx`) so the team can easily track shipping statuses without leaving the table. 

**Code changes implemented:**
- **Replaced plain badges with dropdowns:** We rebuilt the Delivery Status column so that when a status like "In Transit" or "Out for Delivery" is clicked, a small popup card appears.
- **Added tracking info inside the popup:** The card displays the carrier name (e.g., Canada Post) and the full tracking number, along with a **"Copy to Clipboard"** button (using `navigator.clipboard` and `BiCopy` icon).
- **Fixed the "split second" navigation bug:** We added `e.stopPropagation()` to the dropdown click event so clicking the badge doesn't accidentally trigger the row navigation and send the admin to the order detail page.
- **Centered the popup:** We updated the dropdown's CSS positioning from `left-0` to `left-1/2 -translate-x-1/2`, ensuring the popup card appears perfectly centered directly underneath the status badge.
- **Click-outside protection:** We added a `useRef` and a `useEffect` listener so the dropdown automatically closes if you click anywhere else on the screen.

### 2. Google Merchant Center: Custom Product Feed Setup
We bypassed Shopify's flawed automatic product sync and built a brand new, Google-approved product list (feed) directly inside our Next.js website. This guarantees Google only sees clean, valid product data.

**Code changes implemented:**
- **Created a new API route:** Built `/app/api/feed/google/route.ts` (67 lines of code) that pulls all live products using `getAllProducts()` and formats them into Google's required RSS/XML format.
- **Filtered out bad products:** We automatically excluded products with a `$0.00` price or missing images (which previously triggered Google's disapprovals).
- **Fixed the Product ID length issue:** Google has a strict 50-character limit for product IDs. We updated the code to use the short numerical Shopify `p.id` (e.g., `sp-10688490078513`) instead of the longer URL slug, eliminating the error Google reported.
- **Set up caching:** Added a 1-hour cache (`s-maxage=3600`) to the feed so our website servers don't get overwhelmed, while letting Google fetch a fresh list every 24 hours.
- **Deployed & Replaced:** After deploying the updated code, we deleted the broken old Shopify feed inside Google Merchant Center and submitted our new live URL: `https://www.acmevintagesupply.com/api/feed/google`.

**Current status of the 22 products:** They are now officially marked **"Under Review"** (not rejected!), which means Google has accepted the data and is just doing its final checks.
