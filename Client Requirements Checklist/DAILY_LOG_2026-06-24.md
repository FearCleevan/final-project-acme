### 3. Auth Flow Security Hardening

A full audit of the customer sign in and create account flow was done. Six issues were found and fixed across five files:

**What was fixed:**

- **Session secret guard** — The app was silently using a fallback dev secret if `SESSION_SECRET` was missing in production. It now throws a hard error on startup instead, so a misconfigured deployment fails loudly rather than running insecurely.

- **Callback crash fix** — The OAuth callback was calling `JSON.parse()` on Shopify's token response with no error handling. If Shopify returned malformed JSON for any reason, the whole callback would crash with a 500 error. Wrapped in try-catch with a clean redirect to login.

- **Stale session cleanup** — When a token exchange failed, the old OAuth state was left in the session, causing the next login attempt to fail with a "state mismatch" error. Now clears the session on any failure so the next attempt starts fresh.

- **Expired session now clears itself** — When a customer's token expired, the `/api/auth/me` route returned a 401 but left the stale session in the cookie. The client could get stuck in a redirect loop. Now the expired session is destroyed on the 401 so the next login attempt is clean.

- **Logout now awaits** — `session.destroy()` was not being awaited, meaning the session might not actually be cleared before the response was sent. Fixed.

- **Logout client-side safety** — If the logout fetch request to the server fails (network error), the client state is now still cleared. Previously a failed network request would leave the user appearing logged in on the frontend even though the server session may have been destroyed.

- **Stack trace leak fixed** — Error redirects in the authorize and callback routes were exposing the full error message and stack trace in the URL. Now returns a generic error code instead.

- **Shop ID moved to env var** — The Shopify store ID `99152462129` was hardcoded in two route files. Now reads from `NEXT_PUBLIC_SHOPIFY_CUSTOMER_ACCOUNT_ID` which was already set in both `.env.local` and Vercel.

**Commit:** `fix: harden auth flow — session secret guard, callback crash fixes, stale session cleanup`
