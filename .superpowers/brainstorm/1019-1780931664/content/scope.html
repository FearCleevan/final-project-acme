What we're designing

Three connected pieces — let's confirm scope before designing each one

① Account Orders Tab
Logged-in customer views their orders and can submit a cancellation request from their account page — only visible on orders that haven't shipped yet.
Entry point → /account?tab=orders
② Track Order (Guest)
Guest customer tracks their order by order # + email, and can submit a cancellation request the same way — without needing an account.
Entry point → /track-order
③ Admin + Email
Admin sees a cancellation requested flag on the order. Scott manually reviews and approves/rejects. Shopify handles the refund. Customer gets email at each step.
Entry point → /admin/orders/[id]
⚠ Policy context (from Scott's email)
Scott's position: "Not sure if we will accept returns on specialty items." Terms & Conditions already say: "all sales are considered final unless item arrives damaged or misdescribed."

This means cancel ≠ automatic refund. It's a request that Scott reviews — only feasible before the order ships.

Also noted: Returns, Shipping, and FAQ pages don't exist yet — the footer links to them but they 404. These need to be created with Scott's actual answers. This is a separate (simpler) task from the cancel flow.
