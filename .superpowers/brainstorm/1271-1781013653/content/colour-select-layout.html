A — Toggle + Qty Rows

Tap a colour to select it; a quantity row slides in below. Each row shows colour dot, name, stepper, subtotal. CTA shows combined total.

B — Always-visible Table

All colours shown as rows upfront. Qty starts at 0; incrementing it "selects" the colour. Highlighted rows = active. No separate swatch section.

C — Single-select + Opt-in Multi

Default stays single-select (current UX). A "Buying multiple colours?" link reveals a quantity panel. Lowest disruption; multi-buy is opt-in only.

My recommendation: Option A — it's the most natural multi-select pattern for a retail storefront. Swatches feel like toggles, qty rows only appear when needed, and the CTA clearly communicates the combined order. Option B is good for B2B/wholesale. Option C minimises change but hides a useful feature behind a link.
